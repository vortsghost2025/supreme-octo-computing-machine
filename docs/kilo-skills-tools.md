## 📦 dbt MCP SDK Details

The **dbt MCP server** is launched via the command `uvx dbt-mcp`. It relies on the following SDK components:

* **`@kilocode/sdk`** – Core client that reads the `mcpServers` JSON entry and starts a local HTTP server exposing the MCP endpoints.
* **`uv`** – A fast Python package manager used by `uvx` to execute the `dbt-mcp` entry point without a virtual environment.
* **`dbt`** – Your installed dbt CLI (or a Docker image) referenced via the `DBT_PATH` environment variable.

When the server starts, it provides the following endpoints (documented in the dbt MCP source code, but summarised here):

| Endpoint | Function |
|----------|----------|
| `/list_jobs_runs` | Returns recent job runs for a given job ID. |
| `/get_job_run_error` | Returns the full error payload for a specific run. |
| `/list_projects` | Lists dbt projects the user has access to. |
| `/parse` | Runs `dbt parse` to validate the project. |
| `/compile` | Compiles one or more models, returning compiled SQL. |
| `/show` | Executes an ad‑hoc query (`dbt show --inline`). |
| `/test` | Executes a specific test (`dbt test --select`). |
| `/list_models` | Lists models with their dependencies. |

## 🚀 Getting Started with the dbt MCP Server

1. **Install `uv`** (if not already):
   ```bash
   curl -LsSf https://astral.sh/uv/install.sh | sh
   ```
2. **Add your dbt paths** – Edit `.vscode/mcp.local.json` (or copy the placeholder values) to set:
   * `DBT_PROJECT_DIR` – Absolute path to the folder containing `dbt_project.yml`.
   * `DBT_PATH` – Full path to the `dbt` executable (e.g., `C:/Users/.../AppData/Local/Programs/Python/.../Scripts/dbt.exe`).
3. **Start the server** (optional – agents will launch it automatically):
   ```bash
   uvx dbt-mcp
   ```
4. **Verify connectivity** – In Claude Desktop or VS Code, open the MCP server list (`/mcp list` or UI panel). You should see a `dbt` entry.
